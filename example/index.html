Enter As Tab basic sample

This example shows how to use EnterAsTab.js with the default options.




Enter As Tab sample: using tabindex

Instead of using the order of the provided list of elements, you can use a custom order through the tabindex attribute.




Enter As Tab sample: using data-is-action

The default is to consider buttons as an action element, so, although they appear in the list of elements, they are normally activated when they have focus and enter is pressed.

With the autoDetectAction(false) setting, only those that have the data-is-action attribute will be considered as action elements.

Enter As Tab sample: cyclic fields

The default behavior is, when you reach the end of the list of elements, nothing else happens.

However, if you use cyclic(true), when you reach the last field (as long as it is not an action element), the focus will go to the first element in the list.

For this to work, there must be no action elements in the element list passed to the enterastab function.




Enter As Tab sample: select strategy new line

The default behavior in textarea is to advance to the next element.

However, we can change this through the textAreaStrategy directive. It can receive one of 3 values: tab (the default), new line, or ctrl+enter.

new line, maintains the default behavior of the enter key (only within the textarea).

Using ctrl+enter will leave the tab behavior when enter is pressed in the textarea. However, if the ctrl+enter combination is pressed, a new line will be added.
